### 🔄 Команды для разработки

```bash
# Полная пересборка
sudo BUILD_MODE=full ./scripts/build-iso.sh

# Быстрая пересборка (сохраняет chroot)
sudo KEEP_CHROOT=1 BUILD_MODE=full ./scripts/build-iso.sh

# Проверка зависимостей
BUILD_MODE=dry-run ./scripts/build-iso.sh

# Проверка содержимого образа (после сборки)
chmod +x scripts/verify-build.sh
sudo ./scripts/verify-build.sh build/chroot

# Тестирование в QEMU (после сборки)
qemu-system-x86_64 -cdrom build/VibeCodeOS-alpha.iso -m 2048 -enable-kvm
```

### 📝 Заметки для разработчиков

1. **Структура chroot сохраняется** при использовании `KEEP_CHROOT=1` — это экономит ~10-15 минут на каждой пересборке.

2. **Логи сборки** находятся в stdout — рекомендуется перенаправлять в файл для анализа:
   ```bash
   sudo BUILD_MODE=full ./scripts/build-iso.sh 2>&1 | tee build.log
   ```

3. **Отладка в chroot** — можно войти в chroot вручную для тестирования:
   ```bash
   sudo mount -t proc proc build/chroot/proc
   sudo mount -t sysfs sys build/chroot/sys
   sudo mount -o bind /dev build/chroot/dev
   sudo chroot build/chroot /bin/bash
   # ... тестирование ...
   exit
   sudo umount build/chroot/{proc,sys,dev}
   ```

4. **Проблемы с GRUB** — если ISO не загружается, проверьте:
   - Наличие vmlinuz и initrd.img в build/image/boot/
   - Правильность путей в grub.cfg
   - Наличие модулей GRUB в build/image/boot/grub/

1. **Часы в панели MATE**

   - Решение: Добавлен скрипт автонастройки при первом входе
   - Статус: Требует тестирования

2. **GRUB тема не загружается**
   - Проблема: Графическая тема GRUB может не работать
   - Workaround: Используется текстовый режим (terminal_output console)
   - Статус: Не критично для alpha
