### **Dev‑стек VibeCode OS (черновик)**

Этот документ фиксирует целевой стек инструментов разработки и базовые принципы его установки/настройки. Реализация будет постепенно автоматизироваться скриптами в каталоге `scripts/`.

---

### **Терминал и оболочка**

- **Терминал по умолчанию:** `Kitty` (GPU‑ускоренный, быстрый, хорошо кастомизируется).
- **Оболочка:** `Zsh` с `Oh My Zsh` и промптом `Starship`.

План:

1. Скрипт `scripts/setup-shell.sh`:
   - Устанавливает Zsh, Oh My Zsh, Starship.
   - Кладёт готовый `.zshrc` и конфиги Starship.
2. Скрипт `scripts/setup-terminal.sh`:
   - Устанавливает Kitty.
   - Копирует базовые конфиги в `~/.config/kitty`.

---

### **Языки и менеджеры версий**

- **Python:** `pyenv` + несколько версий (3.10/3.11/3.12).
- **Node.js:** `nvm` + одна LTS и одна актуальная версия.
- **Rust:** `rustup` (стабильный toolchain по умолчанию).
- **Go:** последняя стабильная версия из оф. репозитория.
- **Java/Kotlin:** `SDKMAN!` для управления JDK и связанными инструментами.

Планируется скрипт `scripts/setup-langs.sh`, который:

1. Устанавливает менеджеры версий (`pyenv`, `nvm`, `rustup`, `sdkman` и т.д.).
2. Ставит минимальный набор версий для комфортной разработки «из коробки».
3. Добавляет нужные инициализационные строки в shell‑конфиг.

---

### **Редакторы и IDE**

**VSCodium**

- Полностью Open Source аналог VS Code.
- Предустановленные расширения (черновой список):
  - GitLens,
  - Docker,
  - Python,
  - ESLint / Prettier,
  - Rust‑поддержка,
  - поддержка Go.
- Темы:
  - Kilo Code (тема по умолчанию),
  - OpenCode,
  - Catppuccin,
  - Tokyo Night,
  - фирменная тема VibeCode (когда будет готова).

**Neovim**

- Используем готовую конфигурацию: **AstroNvim**.
- Цель — полноценный LSP‑опыт, авто‑дополнение, красивый статусбар и т.д.

**Zed**

- Современный быстрый редактор, поставляется вместе с системой как дополнительный GUI‑редактор.
- Ориентирован на скорость, коллаборацию и «чистый» интерфейс.

---

### **Контроль версий и контейнеры**

- **Git** — по умолчанию в системе.
- **`lazygit`** — TUI‑клиент для Git.
- **Docker** и **Docker Compose**:
  - Установка демона Docker.
  - Добавление текущего пользователя в группу `docker`.
  - Базовый smoke‑тест `docker run hello-world`.

Планируется скрипт `scripts/setup-devtools.sh`, который:

1. Ставит Git (если необходимо) и `lazygit`.
2. Устанавливает Docker и Docker Compose.
3. Выполняет минимальные проверки.

---

### **Автоматизация**

Итоговая цель — возможность выполнить один из следующих сценариев:

- Полная настройка dev‑среды:  
  `./scripts/setup-dev-env.sh`  
  (агрегирует `setup-shell`, `setup-terminal`, `setup-langs`, `setup-devtools`).

- Частичная настройка (по блокам), если пользователь хочет выбирать компоненты.
