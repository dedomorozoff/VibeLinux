### **AI‑стек VibeCode OS (черновик)**

Этот документ описывает целевой набор AI‑инструментов, которые должны работать «из коробки» или устанавливаться в несколько шагов с помощью скриптов.

---

### **Локальные LLM**

- **Ollama**
  - Основной способ запуска локальных LLM‑моделей.
  - План:
    - Установка Ollama (`scripts/ai/install-ollama.sh`).
    - Настройка systemd‑сервиса для автозапуска.
    - Скрипт `scripts/ai/install-ollama-models.sh` для загрузки базовых моделей:
      - `llama3`,
      - `codellama`,
      - `mistral` (и другие по мере необходимости).

- **GUI для локальных моделей: Open WebUI**
  - Запускается локально и подключается к Ollama.
  - Установка и запуск: `scripts/ai/install-open-webui.sh`.

---

### **AI в терминале**

- **`ai-chat`**
  - Простая CLI‑утилита, обращающаяся к локальному Ollama.
  - Позволяет быстро открывать сессию чата прямо в терминале.

Планируется каталог `scripts/ai/` со скриптами:

- `install-terminal-ai.sh` — установка минимальных зависимостей и установка `ai-chat` в систему.
- `ai-chat` — основной entrypoint для общения с моделью из терминала.

---

### **AI в редакторах**

**VSCodium**

- Расширение: **Continue**
  - Поддерживает локальные модели через Ollama.
  - Цель: иметь автодополнение/чат/редактирование с локальным backend без внешних ключей.

**Neovim**

- Плагин для интеграции с локальными LLM через Ollama (будет выбран из совместимых решений).
- Цель: чат и быстрые действия по коду через локальный backend.

---

### **Генерация изображений**

- **Stable Diffusion: ComfyUI**
  - Выбран как основной стек (гибкость пайплайнов, хорошая расширяемость).

План:

1. Скрипт установки ComfyUI: `scripts/ai/setup-comfyui.sh`.
2. Загрузка базовой модели (например, `SD 1.5`) отдельным шагом.
3. Скрипт `scripts/ai/start-sd.sh` для запуска web‑интерфейса.

Особое внимание — к корректной работе с драйверами NVIDIA и CUDA.

---

### **AI‑фреймворки и библиотеки**

В системном Python (отдельное окружение или глобально) планируется предустановить:

- `torch` (CPU‑версия, с инструкцией по установке GPU‑версии),
- `transformers`,
- `langchain`,
- `llama-index`,
- при необходимости другие популярные библиотеки.

Будет подготовлен документ/скрипт:

- `scripts/ai/setup-python-ai-stack.sh` — установка зависимостей.
- `docs/AI-STACK.md` (этот файл) — как точка входа в понимание возможностей.

### **Проприетарные облачные помощники (опционально)**

Эти инструменты **не** входят в базовый офлайн‑набор и требуют учёток/ключей сторонних сервисов, но могут быть установлены отдельными скриптами:

- **OpenAI CLI**
  - Скрипт: `scripts/ai/install-openai-cli.sh`
  - После установки пользователь сам настраивает `OPENAI_API_KEY`.
  - Даёт доступ к моделям OpenAI из терминала (chat/completions и др.).

- **GitHub Copilot CLI**
  - Скрипт: `scripts/ai/install-github-copilot-cli.sh`
  - Требует GitHub‑аккаунт и подписку на Copilot.
  - Позволяет объяснять команды, помогать с Git‑операциями и т.д.

- **Cursor**
  - Проприетарный IDE с глубокой интеграцией AI‑ассистента.
  - Устанавливается из официальных пакетов/инсталлеров; в дистрибутиве даём ссылку и рекомендации по настройке.

- **Kiro**
  - Дополнительный AI‑ассистент/клиент (облачный), установка и использование зависят от текущей платформы и тарифов.

- **Claude (Sonnet и др.)**
  - Используется через веб‑интерфейс или CLI/SDK соответствующего сервиса (Anthropic/партнёры).
  - Требует регистрации и API‑ключа; может быть интегрирован в локальные тулзы по желанию пользователя.

- **Warp**
  - Современный проприетарный терминал с AI‑функциями.
  - Установка из официального репозитория Warp; может использоваться альтернативно системному Kitty, но не является частью базовой поставки.

---

### **Цели по UX AI‑стека**

1. Пользователь сразу понимает, **какие** AI‑инструменты доступны и **как** их запускать.
2. Большинство сценариев — в один клик/одну команду:
   - открыть GUI LLM,
   - начать чат в терминале,
   - активировать ассистента в редакторе.
3. Наличие понятной документации и FAQ по производительности, безопасности и потреблению ресурсов.
