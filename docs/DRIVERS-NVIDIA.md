### NVIDIA в VibeCode OS (черновик для alpha)

Этот документ фиксирует стратегию поддержки проприетарных драйверов NVIDIA на этапе alpha-образа.

---

### 1. Цели для alpha

- **Рабочий сценарий post-install**: пользователь ставит систему, затем одной командой ставит драйвер NVIDIA.
- **Минимальное вмешательство в установщик**: на этапе alpha не модифицируем поведение стандартного установщика Ubuntu (галочка «проприетарные драйверы» остаётся штатной).
- **Фокус на десктопе**: поддержка CUDA/AI-профилей планируется, но не является блокером для первого alpha-ISO.

---

### 2. Базовый сценарий установки (post-install)

После установки VibeCode OS пользователь может выполнить:

```bash
sudo PROFILE=desktop ./scripts/drivers/install-nvidia.sh
```

Это:

- обновит индекс пакетов,
- установит проприетарный драйвер NVIDIA (по умолчанию `nvidia-driver-535`),
- при необходимости предложит перезагрузку.

В будущем будет добавлен профиль:

```bash
sudo PROFILE=ai ./scripts/drivers/install-nvidia.sh
```

для установки того же драйвера + доп. компонентов под AI-нагрузки (CUDA и связанные библиотеки).

---

### 3. Интеграция с ISO и установщиком

- На этапе alpha:
  - ISO не пытается «насильно» ставить проприетарные драйверы.
  - Пользователь может:
    - либо воспользоваться штатной опцией установщика Ubuntu «установить проприетарные драйверы»,
    - либо после установки явно вызвать `scripts/drivers/install-nvidia.sh`.
- После стабилизации базового образа:
  - можно рассмотреть:
    - отдельный профиль установки VibeCode OS с приоритетом на AI (в том числе автодобавление драйвера),
    - интеграцию вызова скрипта после первого входа (через Welcome App или аналог).

---

### 4. Связь с AI-стеком

- AI-скрипты (`scripts/ai/*.sh`) предполагают, что:
  - базовый драйвер NVIDIA уже установлен (через установщик Ubuntu или `install-nvidia.sh`),
  - пользователь понимает требования по GPU/памяти.
- Для GPU-ускоренных сценариев (Ollama с большими моделями, ComfyUI со Stable Diffusion и др.) в доках AI-STECK будет отдельный раздел «Требования к железу и настройка драйверов».

---

### 5. Дальнейшее развитие после alpha

- Поддержка разных веток драйвера (LTS / latest).
- Автовыбор рекомендуемого драйвера по поколению GPU.
- Глубокая интеграция с установщиком и профилями (AI-heavy / Light dev / Student).
- Более тонкая настройка для headless-сценариев и серверов (например, только compute-драйверы без графического стека).
