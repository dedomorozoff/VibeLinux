### Руководство по тестированию VibeCode OS alpha

Этот документ описывает процесс тестирования alpha-образа VibeCode OS.

---

### Подготовка к тестированию

#### Требования

- **Для виртуальной машины:**
  - VirtualBox 7.0+ или QEMU/KVM
  - 2+ GB RAM
  - 20+ GB свободного места на диске (для установки)

- **Для реального железа:**
  - USB-флешка 4+ GB
  - Компьютер с поддержкой UEFI/Legacy BIOS
  - Рекомендуется: отдельный диск для установки

#### Создание загрузочной флешки

```bash
# Linux
sudo dd if=build/VibeCodeOS-alpha.iso of=/dev/sdX bs=4M status=progress && sync

# Windows (используйте Rufus или Etcher)
# macOS (используйте Etcher или команду dd)
```

---

### Тестирование в VirtualBox

#### 1. Создание виртуальной машины

```bash
# Создаём VM
VBoxManage createvm --name "VibeCodeOS-Test" --ostype Ubuntu_64 --register

# Настраиваем параметры
VBoxManage modifyvm "VibeCodeOS-Test" \
  --memory 2048 \
  --vram 128 \
  --cpus 2 \
  --boot1 dvd \
  --boot2 disk \
  --nic1 nat \
  --audio-driver pulse \
  --audiocontroller hda

# Создаём виртуальный диск (для тестирования установки)
VBoxManage createhd --filename ~/VirtualBox\ VMs/VibeCodeOS-Test/disk.vdi --size 20480

# Подключаем диск и ISO
VBoxManage storagectl "VibeCodeOS-Test" --name "SATA" --add sata --controller IntelAhci
VBoxManage storageattach "VibeCodeOS-Test" --storagectl "SATA" --port 0 --device 0 --type hdd --medium ~/VirtualBox\ VMs/VibeCodeOS-Test/disk.vdi
VBoxManage storageattach "VibeCodeOS-Test" --storagectl "SATA" --port 1 --device 0 --type dvddrive --medium build/VibeCodeOS-alpha.iso

# Запускаем
VBoxManage startvm "VibeCodeOS-Test"
```

Или через GUI:
1. Создать новую VM (Type: Linux, Version: Ubuntu 64-bit)
2. RAM: 2048 MB
3. Создать виртуальный диск: 20 GB
4. Подключить ISO в настройках Storage
5. Запустить

#### 2. Тестирование в QEMU

```bash
# Быстрый запуск для тестирования Live-режима
qemu-system-x86_64 \
  -cdrom build/VibeCodeOS-alpha.iso \
  -m 2048 \
  -enable-kvm \
  -cpu host \
  -smp 2

# С виртуальным диском для тестирования установки
qemu-img create -f qcow2 vibecode-test.qcow2 20G
qemu-system-x86_64 \
  -cdrom build/VibeCodeOS-alpha.iso \
  -hda vibecode-test.qcow2 \
  -m 2048 \
  -enable-kvm \
  -cpu host \
  -smp 2 \
  -boot d
```

### Чек-лист тестирования Live-режима

#### Базовая загрузка
- [ ] ISO загружается в GRUB
- [ ] Выбор пункта меню "VibeCode OS (Live)" работает
- [ ] Система загружается до рабочего стола MATE
- [ ] Autologin работает (пользователь `vibecode`)
- [ ] Время загрузки приемлемое (< 2 минут в VM)

#### Desktop Environment
- [ ] Панель MATE отображается
- [ ] **Часы в панели показывают правильное время**
- [ ] Меню Applications работает
- [ ] Меню System работает
- [ ] Переключение между приложениями работает (Alt+Tab)
- [ ] Рабочий стол отзывчивый, нет зависаний

#### Файловый менеджер (Caja)
- [ ] Caja запускается
- [ ] Можно открыть домашнюю директорию
- [ ] Можно создать файл/папку
- [ ] Можно удалить файл/папку
- [ ] Можно переименовать файл/папку
- [ ] Контекстное меню работает

#### Терминал
- [ ] Konsole запускается
- [ ] Можно вводить команды
- [ ] Базовые команды работают (ls, pwd, echo)
- [ ] Копирование/вставка работает
- [ ] Можно открыть несколько вкладок

#### Сеть
- [ ] Сетевой апплет в панели показывает подключение
- [ ] Ethernet подключение работает (если доступно)
- [ ] Wi-Fi подключение работает (если доступно)
- [ ] `ping 8.8.8.8` работает
- [ ] `ping google.com` работает (DNS)
- [ ] Браузер может открыть веб-страницы (если установлен)

#### Звук
- [ ] Звуковой апплет в панели отображается
- [ ] Можно изменить громкость
- [ ] Тестовый звук воспроизводится (если есть аудио-файл)

#### Локализация
- [ ] Интерфейс на русском языке
- [ ] Раскладка клавиатуры переключается (если настроено)
- [ ] Русские символы отображаются корректно

#### Производительность
- [ ] Использование RAM: _____ MB (записать значение)
- [ ] Загрузка CPU в idle: _____ % (записать значение)
- [ ] Система отзывчива, нет лагов

### Чек-лист тестирования установки

#### Запуск установщика
- [ ] Ярлык установщика присутствует на рабочем столе
- [ ] Установщик (ubiquity) запускается
- [ ] Можно выбрать язык установки
- [ ] Можно выбрать раскладку клавиатуры

#### Процесс установки
- [ ] Можно выбрать диск для установки
- [ ] Разметка диска работает (автоматическая)
- [ ] Разметка диска работает (ручная)
- [ ] Можно создать пользователя
- [ ] Можно установить пароль
- [ ] Установка завершается без ошибок
- [ ] Время установки: _____ минут (записать)

#### После установки
- [ ] Система предлагает перезагрузку
- [ ] После перезагрузки система загружается с диска
- [ ] GRUB показывает меню загрузки
- [ ] Система загружается до экрана входа
- [ ] Можно войти с созданным пользователем
- [ ] Все функции Live-режима работают после установки

### Тестирование драйверов

#### VirtualBox Guest Additions
- [ ] Разрешение экрана изменяется автоматически
- [ ] Общий буфер обмена работает
- [ ] Drag & Drop работает (если настроено)
- [ ] Shared Folders доступны (если настроены)

#### NVIDIA драйверы (на реальном железе)
- [ ] Скрипт `install-nvidia.sh` запускается
- [ ] Драйверы устанавливаются без ошибок
- [ ] После перезагрузки `nvidia-smi` работает
- [ ] GPU определяется корректно

### Отчёт о проблемах

При обнаружении проблем заполните следующую информацию:

```
Проблема: [краткое описание]
Шаги воспроизведения:
1. [шаг 1]
2. [шаг 2]
3. ...

Ожидаемое поведение: [что должно было произойти]
Фактическое поведение: [что произошло]

Окружение:
- Тип: [VM/Реальное железо]
- VM: [VirtualBox/QEMU/другое]
- RAM: [количество]
- CPU: [модель]
- GPU: [модель, если применимо]

Логи/Скриншоты: [приложить, если есть]
```

### Сбор логов для отладки

```bash
# Системные логи
journalctl -b > system.log

# Логи X сервера
cat ~/.xsession-errors > xsession.log

# Информация о системе
neofetch > system-info.txt
lsb_release -a >> system-info.txt
uname -a >> system-info.txt

# Список установленных пакетов
dpkg -l > packages.txt

# Информация о дисках
lsblk > disks.txt
df -h >> disks.txt
```
